## REST API
The POSTman documentation of the REST API can be found @ https://documenter.getpostman.com/view/3653795/SVfWN6KS

## REST API Limits
Most of our REST API endpoints are rate limited. There are two levels of rate limiting in place:
- IP based limiting (by cloudflare)
  - At most 500 calls per 10 seconds
- User based limiting, at most:
  - 1 call per 200 ms: `/orders/create`, `/orders/modify`
  - 1 call per second: `/swap`, `/swap/confirm/:swapId`, `/oauth/token`, `/oauth/redirect-uri`
  - 1 call per 5 seconds: `/users/wallets/balance-history`, `/ignium/transactions`, `/orders/history`, `/fills`, `/users/wallets/history`
  - 20 calls per 1 minute: `/orders/create`, `/orders/modify`
  - 100 calls per 1 minute: `/book`, `/candles`, `/ticks`
  - 1 call per 1 minute: `/balances`, `/webhook/test`
  - 300 calls per 1 hour: `/orders/create`, `/orders/modify`
  - 1000 calls per 1 day: `/orders/create`, `/orders/modify`

## Roundings and multipliers
The prices and quantities in the book are rounded to a finite number of digits.
- Most current roundings for all assets: https://api.coinmetro.com/assets
- Most current roundings for all pairs: https://api.coinmetro.com/markets

## WebSockets
The WebSockets can be accessed at:
- `wss://api.coinmetro.com/ws` (live environment)
- `wss://api.coinmetro.com/open/ws` (demo environment)

The connection supports two arguments in the query string:
- `pairs`: Comma separated list of pairs for which book update subscription is requested
- `token`: JWT token generated through one of the login paths

Both parameters are optional.

Example: `wss://api.coinmetro.com/ws?pairs=BTCEUR,LTCEUR`

The WebSockets emit the following messages:

### Order Status Message (authenticated only)
```json
{
  "orderStatus": {
    "orderType": "market | limit",
    "buyingCurrency": "string",
    "sellingCurrency": "string",
    "sellingQty": "number",
    "userID": "string",
    "orderID": "string",
    "timeInForce": "number (1=GTC,2=GTD,3=IOC,4=FOK)",
    "boughtQty": "number",
    "soldQty": "number",
    "creationTime": "number",
    "seqNumber": "number",
    "firstFillTime": "number (ms since epoch)",
    "lastFillTime": "number (ms since epoch)",
    "fills": [
      {
        "seqNumber": "number",
        "timestamp": "number (ms since epoch)",
        "qty": "number",
        "price": "number",
        "side": "buy | sell"
      }
    ],
    "completionTime": "number (ms since epoch)",
    "takerQty": "number"
  }
}
```

### Wallet Update Message (authenticated only)
```json
{
  "walletUpdate": {
    "walletId": "string",
    "currency": "string",
    "label": "string",
    "userId": "string",
    "description": "string",
    "amount": "number",
    "JSONdata": {
      "price": "string (e.g. '6730.17 BTC/EUR')",
      "fees": "number",
      "notes": "string"
    },
    "timestamp": "string (ISODate)",
    "balance": "number"
  }
}
```

### Book Update message (subscribed pairs only)
Requires `pairs` query parameter. Every update includes a CRC32 `checksum`.
The checksum string is generated by concatenating sorted non-null asks + sorted non-null bids.

```
CRC32(
  Object.keys(book.ask).filter(p => book.ask[p]).sort().map(p => `${p}${book.ask[p]}`).join("") +
  Object.keys(book.bid).filter(p => book.bid[p]).sort().map(p => `${p}${book.bid[p]}`).join("")
)
```

**Format:**
```json
{
  "bookUpdate": {
    "pair": "string",
    "seqNumber": "number",
    "ask": {
      "price_string": "qty_delta (number)"
    },
    "bid": {
      "price_string": "qty_delta (number)"
    },
    "checksum": "number"
  }
}
```
Keys are price levels, values are quantity **deltas** in the base currency. A value of 0 means the level should be removed.

### Tick message (all pairs, always)
Every tick also includes the most current **ask** and **bid**.

```json
{
  "tick": {
    "pair": "string",
    "price": "number",
    "qty": "number",
    "timestamp": "number (ms since epoch)",
    "seqNum": "number",
    "ask": "number",
    "bid": "number"
  }
}
```

**NOTE**: Live WS tick messages have `qty: 1e-12` (synthetic). Real trade quantities are only available via REST `/exchange/ticks/{pair}`.

### New Referral message (authenticated only)
```json
{
  "newReferral": {
    "tier": "number (1 or 2)",
    "refId": "string"
  }
}
```

## REST Endpoints (Verified 2026-02)

### Products/Prices
- `GET /exchange/prices` → `{latestPrices: [{pair, timestamp, price, qty, base, quote, ask, bid}]}`
- 148 products as of Feb 2026

### Tick History (Trade-like data)
- `GET /exchange/ticks/{PAIR}` → `{tickHistory: [{pair, price, qty, timestamp, seqNum}]}`
- Real quantities (unlike WS ticks), but **NO side field**
- Rate limit: 100 calls per 1 minute

### Orderbook (REST)
- `GET /book/{PAIR}` → **404 as of Feb 2026 (deprecated)**
- Only WS `bookUpdate` provides orderbook data

### Assets/Markets Metadata
- `GET /assets` → asset roundings
- `GET /markets` → pair roundings and multipliers
